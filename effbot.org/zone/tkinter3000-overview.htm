Tkinter 3000

The Tkinter 3000 project attempts to create a better (faster, smaller) Tk interface, and provide ways to extend the Tkinter library with new widgets, geometry managers, and more. The new design is mostly compatible with Tkinter, as shipped with Python 1.5.2.

This site contains development versions, and related documentation.

Introduction

Tkinter 3000's Python interface is based on Python 1.5.2's Tkinter layer. Like Tkinter, it provides:

  1. A component-oriented layer on top of the Tk library. Each widget type is represented by a class.

  2. Separation between views (widgets) and controllers (event bindings). Each widget instance can inherit bindings from one or more controller classes.

  3. Separation between views and geometry management. Any widget can act as a geometry manager for any number of other widgets.

In addition to the built-in widgets, Tkinter 3000 provides generic widget and controller classes (the Widget Construction Kit, or WCK) that you can use to implement your own custom widgets.
